# Invoice System API Documentation

This document provides a detailed overview of all endpoints available in the Invoice module.

## 1. Single (Individual) Invoices

### List Invoices
- **URL**: `GET /api/v1/invoices/`
- **Role**: `Any active user`
- **Purpose**: Retrieve a paginated list of all individual order invoices.
- **Parameters**: `skip` (default 0), `limit` (default 100).

### Create Manual Invoice
- **URL**: `POST /api/v1/invoices/`
- **Role**: `Any active user`
- **Purpose**: Generate an invoice from a specific Order.
- **Rules**: Subtotal is derived strictly from line items.
- **Key Body Fields**:
  - `customer_id`: int
  - `order_id`: int
  - `discount_percentage`: float (Applied on subtotal)
  - `due_date`: datetime

### Get Invoice Detail
- **URL**: `GET /api/v1/invoices/{invoice_id}`
- **Role**: `Any active user`
- **Purpose**: Returns full details of a specific invoice.

### Update Invoice
- **URL**: `PUT /api/v1/invoices/{invoice_id}`
- **Role**: `Staff` or `Admin`
- **Purpose**: Update status, PDF URLs, or Stripe IDs.
- **Note**: Cannot update invoices marked as "paid".

### Delete (Cancel) Invoice
- **URL**: `DELETE /api/v1/invoices/{invoice_id}`
- **Role**: `Superuser`
- **Purpose**: Marks an invoice as "cancelled" (soft delete).

---

## 2. Combined Invoices (Statements)

### Generate Single Statement
- **URL**: `POST /api/v1/invoices/generate-daily-combined`
- **Role**: `Staff` or `Admin`
- **Purpose**: Aggregates all unpaid, uncombined invoices for a specific customer into one statement.
- **Query Parameters**:
  - `customer_id`: int (Required)
  - `discount_percentage`: float (Admin override for the whole statement)
  - `due_date`: datetime (Optional override)

### Bulk Generate Statements
- **URL**: `POST /api/v1/invoices/auto-generate-all`
- **Role**: `Staff` or `Admin`
- **Purpose**: Scans all B2B customers and generates statements based on their configured billing cycles.

### List Statements
- **URL**: `GET /api/v1/invoices/combined`
- **Role**: `Any active user` (Partners see only their own)
- **Purpose**: Retrieve a list of all combined invoices.

### Get Statement Detail
- **URL**: `GET /api/v1/invoices/combined/{combined_id}`
- **Role**: `Any active user`
- **Purpose**: Returns high-level statement data and the list of linked sub-invoices.

---

## 3. Payments & Actions

### Record Single Payment
- **URL**: `POST /api/v1/invoices/{invoice_id}/pay`
- **Role**: `Any active user`
- **Purpose**: Records a manual payment, updates customer debt balance, and changes status to "paid".

### Record Statement Payment
- **URL**: `POST /api/v1/invoices/combined/{combined_id}/pay`
- **Role**: `Staff` or `Admin`
- **Purpose**: Pays the entire statement at once. This automatically marks **ALL** linked invoices as "paid".

### Send via WhatsApp
- **URL**: `POST /api/v1/invoices/{invoice_id}/send-whatsapp`
- **Role**: `Any active user`
- **Purpose**: Triggers a WhatsApp notification to the customer with the PDF link.

---

## 4. Calculation Logic (ERP-Grade)

- **One Source of Truth**: Statement subtotals are always the sum of linked individual invoice `amount_due` fields.
- **Safety Guards**: All PDF generation is protected by strict math assertions. If `total != subtotal - discount`, the generation aborts.
---

## 5. Invoice Status Lifecycle

Understanding the status of an invoice or statement is critical for financial auditing.

| Status | Definition | Trigger / Action |
| :--- | :--- | :--- |
| **`draft`** | **Initial State**. The record is created in the DB and PDF is generated, but it is NOT yet shared with the customer. | Auto-generated by scheduler or manual creation. |
| **`sent`** | **Issued**. Customer has been notified. This is now an active Account Receivable. | Calling the `send-whatsapp` or notification endpoint. |
| **`paid`** | **Settled**. Funds have been received. Credit line is restored for the customer. | Using the `/pay` endpoint (manual or statement-level). |
| **`cancelled`** | **Voided**. The invoice was created in error or the order was returned/rejected. | Calling the `DELETE` endpoint on an unpaid invoice. |

> [!NOTE]
> Paid invoices **cannot** be updated or deleted. You must issue a credit note or manual adjustment if a correction is needed after payment.
